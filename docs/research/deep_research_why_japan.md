## 各プロジェクトについて調べること

各プロジェクトについて、以下の観点から**簡潔に**（100-150文字程度）日本との関わりを説明してください：

### 1. 開発者・組織
- 日本人の主要開発者は誰か？
- 日本企業・大学・研究機関が主導しているか？
- 例：「まつもとゆきひろ氏が開発」「Preferred Networks主導」「京大×NTT共同開発」

### 2. 開発の経緯
- 日本で生まれたプロジェクトか？
- 日本の課題解決のために作られたか？
- 例：「日本語処理の課題を解決」「2011年震災時に日本のOSMコミュニティが立ち上げ」

### 3. 日本コミュニティの活動
- 日本での普及・活用状況
- Code for Japanなどの関与
- 例：「日本での導入事例が多数」「Code for Japanが公式パートナー」

### 4. 日本独自の貢献
- 日本版の作成・ローカライズ
- 日本向けの機能追加
- 例：「日本向けにGTFS拡張」「日本語ドキュメント整備」

## アウトプット形式

以下のJSON形式で、各プロジェクトの`why_japan`フィールドを埋めてください：

```json
{
  "Optuna": "Preferred Networksが開発したハイパーパラメータ最適化ライブラリ。日本発のMLツールとして世界中で利用されている。",
  "Misskey": "syuilo（篠田英治）氏が2014年に開発を開始した日本発のActivityPub分散SNS。日本のFediverse文化を牽引。",
  "Fluentd": "古橋貞之氏らTreasure Data出身の日本人開発者が創始。CNCF卒業プロジェクトとして国際的に認知される日本発OSS。",
  ...
}
```

### 記述のポイント

1. **簡潔に**（100-150文字程度）
2. **具体的な人名・組織名**を含める
3. **日本との関わりが明確**になるように
4. **実績・認知度**があれば記載（例：「CNCF卒業」「全国○○自治体で採用」）
5. 専門用語は適度に、一般の人にも分かるように

### 悪い例・良い例

❌ 悪い例（抽象的）:
```
"日本で開発されたプロジェクトで、多くの人に使われています。"
```

✅ 良い例（具体的）:
```
"まつもとゆきひろ氏が「書いていて楽しい」を重視して開発した日本発のプログラミング言語。Ruby on Railsなど世界中で利用される。"
```

## 情報源の例

以下のような情報源を優先的に調査してください：

1. **公式ドキュメント・About**ページ
2. **GitHubのREADME**（Contributorsセクション）
3. **企業のプレスリリース**（PFN、Sony、Treasure Dataなど）
4. **財団の発表**（CNCF、Apache Softwareなど）
5. **開発者のブログ・インタビュー**
6. **学術論文**（特にNLP、音声認識系）
7. **Code for Japanの活動記録**
8. **Wikipedia**（日本語版・英語版）

---

## 注意事項

1. **推測は避ける**: 明確な情報源がない場合は「情報不明」と記載
2. **正確な人名・組織名**: カタカナ表記も含めて正確に
3. **時制に注意**: 過去のプロジェクトは「〜だった」、現在進行形は「〜している」
4. **敬称不要**: 「氏」は付けてOKだが「さん」は不要
5. **事実ベース**: 「素晴らしい」「画期的」などの主観的評価は避ける

## 質問・不明点がある場合

もし特定のプロジェクトについて信頼できる情報が見つからない場合は、以下のように記載してください：

```json
{
  "プロジェクト名": "【要確認】〇〇の可能性があるが、公式情報が見つからず詳細不明。"
}
```

この場合、可能性がある情報源も併せて教えてください。
